hw3內容：從一個data source (i.e. one more multiple spreadsheet(s)) 以及使用 AI coding tool (i.e. Cursor) 來打造一個純前端的網路服務。
- 程式語言除了必要的 HTML, CSS 之外，動態網頁的部分規定使用 React.ts (i.e. TypeScript 版本的 React)，而不是 pure JavaScript or React.js。
- UI 的部分請使用現成的框架 (i.e. 不要只是使用 pure CSS)，你可以選擇課堂上有教過的 MaterialUI, Ant Design, Shadcn, TailwindCSS (with cn), 或者是任何你喜歡的框架。
- Spreadsheet 原始應為一個或者是多個 .csv 檔案，存放在 hw3/src/public/data 底下。你的服務應該要做到當使用者編輯/更改 .csv 檔案，更新到你的 repo 以後，你的服務應該會自動更新 (either hot reload, or restart app)，不需要改動其他的程式碼

🧠 核心功能設計
1️⃣ 舞團瀏覽頁（首頁）
目標：能快速找到有興趣的 cover 團
主要功能
搜尋框（團名或歌名）
篩選器：地區（雙北 / 桃園 / 台中 / 高雄）
每個團卡片顯示：
團名 - 歌名（含連結）
缺的位置（例如「缺 Ella、Gawon」）
練習時間與地點
拍攝時間與地點
點擊展開詳情
顯示規則與完整說明
「💾 儲存」按鈕（加入清單）
2️⃣ 我的行事曆頁（Calendar View）
目標：模擬加入多個團後的練習衝突情況
主要功能
顯示所有「已儲存團」的練習與拍攝時間
可勾選/取消某個團 → 即時更新日曆顯示
點擊活動可以看到：
時間、地點
所屬團體名稱
是否為練習/拍攝
若時間重疊 → 以紅色標註提醒衝突
3️⃣ 儲存清單 / 申請頁
目標：統一管理想申請的 cover 團
主要功能
顯示所有已儲存的團（清單格式）
「回首頁」 / 「新增更多申請」按鈕
「送出申請」 → 模擬發出申請（此時鎖定，不可再取消）
顯示「等待團主回覆中」狀態
🖼️ 使用流程範例
1️⃣ 使用者打開首頁，搜尋「ZEROBASEONE」
2️⃣ 發現幾個徵人的團（顯示缺誰、練習地點）
3️⃣ 點擊「💾 儲存」 → 新增到清單
4️⃣ 開啟行事曆 → 勾選幾個團，看是否撞時間
5️⃣ 決定要申請其中三個 → 點「送出申請」
6️⃣ 顯示「等待團主回覆中」
